    NetCDF MDArray metadata: corrected plan and agent prompt#

    This document updates the plan to add “read all metadata” capabilities for NetCDF files using GDAL’s Multidimensional (MDim) API, aligned with the current repository structure and existing functionality.

    Key facts about current code: - The project uses a src/ layout. Relevant paths are under: src\pyramids\netcdf. - src\pyramids\netcdf\netcdf.py defines class NetCDF(Dataset). It already supports: - Opening NetCDF as MDArray-backed datasets via GDAL - Access to variables through get_variable() and names via get_variable_names() - Opening files through NetCDF.read_file(path, open_as_multi_dimensional=True) - Dimension parsing exposed via self.meta_data which is a MetaData object - src\pyramids\netcdf\dimensions.py implements MetaData, DimensionsIndex, and helpers to parse: - GDAL “classic” netCDF metadata keys: NETCDF_DIM_* - Per-dimension attributes ("<name>#<attr>") like time#units, lat#axis, etc. - There is currently no traversal of the full GDAL MDim tree (groups, arrays, attributes) beyond the dimension/attribute convenience provided by dimensions.py.

    Goal of this plan: Add a new, additive capability to enumerate and normalize ALL metadata (groups, arrays, dimensions, attributes, and convenience fields) when a NetCDF is opened as MDArray, without breaking existing APIs.

    Public API surface (additive, aligned with repo)#

    Files to create or modify (Windows paths): - Create: src\pyramids\netcdf\models.py (data models) - Create: src\pyramids\netcdf\metadata.py (reader/serializer utilities) - Modify: src\pyramids\netcdf\netcdf.py (add one convenience method) - Optional CLI: src\pyramids\cli\netcdf_metadata.py (only if CLI is desired)

    New functions/classes: - In src\pyramids\netcdf\metadata.py: - get_mdim_metadata(source: str | "NetCDF", open_options: dict | None = None) -> NetCDFMetadata - Preferred: open with NetCDF.read_file(path, open_as_multi_dimensional=True) when source is a path. - If source is an existing NetCDF instance, reuse its _raster handle and file_name. - Traverse the root Group recursively; collect global attributes, groups, arrays, and dimensions. - Normalize values to be JSON-serializable. - to_json(metadata: NetCDFMetadata) -> str - from_json(s: str) -> NetCDFMetadata - to_dict(metadata: NetCDFMetadata) -> dict[str, Any] - flatten_for_index(metadata: NetCDFMetadata) -> dict[str, Any] (flat fields for catalog/search)

    • In src\pyramids\netcdf\models.py (frozen dataclasses):
    • AttributeValue = bool | int | float | str | list[bool|int|float|str]
    • ArrayInfo
    • GroupInfo
    • DimensionInfo
    • StructuralInfo

    • NetCDFMetadata Notes: keep all fields JSON-serializable without custom encoders.

    • In src\pyramids\netcdf\netcdf.py (class NetCDF):

    • Add: def get_all_metadata(self, open_options: dict | None = None) -> "NetCDFMetadata":
      • Delegate to mdim_metadata.get_mdim_metadata(self, open_options).
      • Do not remove or change meta_data property (dimension-focused view stays intact).

    Data model (models.py)#

    Use frozen dataclasses. Naming mirrors GDAL MDim concepts and CF conventions where helpful.

    • class NetCDFMetadata:
    • driver: str
    • root_group: str | None
    • groups: dict[str, GroupInfo] (keyed by full group name, e.g., "/", "/subgrp")
    • arrays: dict[str, ArrayInfo] (keyed by full array name, e.g., "/var", "/subgrp/var2")
    • dimensions: dict[str, DimensionInfo] (keyed by full dim name)
    • global_attributes: dict[str, AttributeValue]
    • structural: StructuralInfo | None
    • open_options_used: dict[str, str] | None
    • created_with: dict[str, str] (e.g., {"library": "GDAL", "version": gdal.version})

    • dimension_overview: dict[str, Any] | None (see Integration with MetaData)

    • class GroupInfo:

    • name: str (short name)
    • full_name: str (e.g., "/subgrp")
    • attributes: dict[str, AttributeValue]
    • children: list[str] (full names)

    • arrays: list[str] (full array names)

    • class ArrayInfo:

    • name: str
    • full_name: str
    • dtype: str (GDAL data type name)
    • shape: list[int]
    • dimensions: list[str] (full dim names or short names, but be consistent)
    • attributes: dict[str, AttributeValue]
    • unit: str | None (MDArray.GetUnit())
    • nodata: int|float|str|None (from GetNoDataValue* and/or CF attributes)
    • scale: float | None (GetScale() or CF scale_factor)
    • offset: float | None (GetOffset() or CF add_offset)
    • srs_wkt: str | None (from GetSpatialRef())
    • srs_projjson: str | None
    • coordinate_variables: list[str]
    • structural_info: dict[str, str] | None (GetStructuralInfo())

    • block_size: list[int] | None (GetBlockSize())

    • class DimensionInfo:

    • name: str
    • full_name: str
    • size: int
    • type: str | None (if available)
    • direction: str | None (if available)

    • indexing_variable: str | None (Dimension.GetIndexingVariable())

    • class StructuralInfo:

    • driver_name: str
    • driver_metadata: dict[str, str] | None

    AttributeValue should normalize GDAL Attribute.Read* outputs; lists must preserve order.

    Traversal & extraction rules (GDAL MDim API)#

    Open the dataset through the library’s entrypoint, then traverse with GDAL: - Open using NetCDF.read_file(path, open_as_multi_dimensional=True) to ensure MDArray mode. - Get the root group via nc._raster.GetRootGroup(). - Enumerate groups/arrays/dimensions using Group.GetGroupNames(), Group.OpenGroup(name), Group.GetMDArrayNames(), Group.OpenMDArray(name), Group.GetDimensions(). - Attributes on groups and arrays: GetAttributes() then read each via Attribute.Read() or specialized ReadAsInt*, ReadAsDouble*, ReadAsString* including vector variants; ensure JSON-ready types (no numpy scalars). - Arrays: GetUnit(), GetNoDataValueAsDouble/AsInt64/AsString (check what is available), GetScale(), GetOffset(), GetSpatialRef() -> export both WKT and PROJJSON if available. - Arrays: GetStructuralInfo() -> convert to dict; GetBlockSize() -> list[int] when present. - Arrays: GetCoordinateVariables() -> list names; also list GetDimensions() and GetShape(). - Dimensions: from Group.GetDimensions() and Dimension.GetIndexingVariable(); capture size and optional kind/direction if exposed by GDAL. - Capture driver-level translations (CF): _FillValue / missing_value precedence for nodata; scale_factor / add_offset for scale/offset. - Persist any open options used.

    Normalization & edge cases: - Scalar and vector attributes; decode bytes to str (UTF-8). - Do not read data values. - Handle nested groups, scalar/string arrays, empty attributes gracefully. - No xarray/netCDF4 dependency.

    Integration with dimensions.MetaData (existing functionality)#

    Re-use the existing dimension parser for a concise, CF-friendly summary: - For a NetCDF instance nc, obtain nc.meta_data (instance of dimensions.MetaData). - Include a JSON-friendly snapshot in NetCDFMetadata.dimension_overview with the following shape: - names: list[str] - sizes: dict[str, int] (dimension size by name) - attrs: dict[str, dict[str, str]] (per-dimension attributes like time#units) - values: dict[str, list[int|float|str]] | None (dimension coordinate values when available) - Build this snapshot by calling MetaData.from_metadata(nc._raster.GetMetadata()) and serializing its dims and attrs into the structure above. - This is additive to the full MDim dimensions map gathered from GDAL; it provides a convenient, backward-compatible view.

    Tests (pytest)#

    Add new tests alongside existing suite using current structure: - Location: tests\\netcdf\\test_mdim_metadata.py - Tiny handcrafted NetCDF fixtures under tests\\data\\netcdf (or reuse existing small files if suitable). - Parametrized cases: - Minimal NetCDF with 1 group, 1 array, scalar attributes - Arrays with _FillValue, missing_value, both (precedence) - Arrays with scale_factor/add_offset - Coordinate variables: (time, lat, lon) - Nested groups with attributes - CRS present/absent (grid_mapping captured as raw attribute when SRS missing) - Structural info present/absent - Mixed attribute types (ints, floats, strings, vectors) - Shape/dimensions alignment - Open options propagation - In all tests, open files via NetCDF.read_file(path, open_as_multi_dimensional=True) and call nc.get_all_metadata().

    Each test should have clear docstrings and assert JSON round-trip via to_json/from_json.

    Quality gates / acceptance criteria#

    • Any valid NetCDF opens and returns populated Metadata.
    • All attributes captured and normalized; no silent loss.
    • Convenience fields (nodata, scale, offset, unit, CRS, structural) filled when available.
    • JSON round-trip preserves information (structural and attribute data).
    • Good test coverage for the new module; existing tests continue to pass.
    • No runtime dependency on xarray/netCDF4.

    Agentic workflow: full prompt to implement the feature#

    You are given a Python project with GDAL-based NetCDF support. Implement an additive feature to read and normalize ALL metadata from NetCDF files using the GDAL Multidimensional (MDim) API. Follow these instructions precisely.

    Context: - OS: Windows - Python package uses src/ layout. - Existing code parses dimension metadata via src\pyramids\netcdf\dimensions.py and exposes it in NetCDF.meta_data. Do NOT remove or break this.

    Goals: 1) Create new models and reader utilities to traverse GDAL MDim groups/arrays/dimensions and collect normalized metadata. 2) Add a convenience method on NetCDF to expose this metadata for a file already opened in the library. 3) Provide JSON serialization helpers, a small doc page, and tests.

    Files and APIs to add: - Create src\\pyramids\\netcdf\\models.py with frozen dataclasses: AttributeValue, ArrayInfo, GroupInfo, DimensionInfo, StructuralInfo, NetCDFMetadata (see the Data model section above for exact fields and types). - Create src\\pyramids\\netcdf\\metadata.py with functions: - get_mdim_metadata(source: str | NetCDF, open_options: dict | None = None) -> NetCDFMetadata - to_json(metadata: NetCDFMetadata) -> str - from_json(s: str) -> NetCDFMetadata - to_dict(metadata: NetCDFMetadata) -> dict[str, Any] - flatten_for_index(metadata: NetCDFMetadata) -> dict[str, Any] - Modify src\\pyramids\\netcdf\\netcdf.py to add: - def get_all_metadata(self, open_options: dict | None = None) -> "NetCDFMetadata": from .mdim_metadata import get_mdim_metadata return get_mdim_metadata(self, open_options)

    Implementation details: - Always open files through NetCDF.read_file(path, open_as_multi_dimensional=True) in implementation and tests; or accept an existing NetCDF instance. - Use GDAL’s MDim Python API to enumerate groups (Group.GetGroupNames/OpenGroup), arrays (GetMDArrayNames/OpenMDArray), dimensions (GetDimensions), and attributes (GetAttributes + Attribute.Read or ReadAs variants) starting from nc._raster.GetRootGroup(). - For MDArray convenience fields: use GetUnit, GetNoDataValue* (resolve CF _FillValue/missing_value precedence), GetScale, GetOffset, GetSpatialRef (export WKT and PROJJSON if available), GetStructuralInfo (dict), GetBlockSize (list[int]), GetCoordinateVariables. - Ensure all values are JSON-serializable (convert numpy scalars to Python types, bytes to UTF-8 strings). - Persist open options used; add created_with info using GDAL version. - Do not read array data values; only metadata. - Populate dimension_overview in NetCDFMetadata by serializing nc.meta_data into a JSON-friendly dict with fields: names, sizes, attrs, values.

    Tests: - Add tests at tests\\netcdf\\test_mdim_metadata.py covering the scenarios listed in the Tests section. - Open files with NetCDF.read_file(...), call nc.get_all_metadata(), and validate dimension_overview matches nc.meta_data. - Include JSON round-trip tests via to_json/from_json. - Reuse or add tiny NetCDF fixtures under tests\\data\\netcdf.

    Docs: - Add docs\\reference\\netcdf_metadata.md with usage examples and notes.

    Acceptance: - New tests pass; existing tests remain green. - Calling NetCDF.get_all_metadata() returns a populated NetCDFMetadata (including dimension_overview) for typical files under tests\\data\\netcdf and examples.

    Non-goals: - Do not add xarray/netCDF4 dependencies. - Do not change existing dimension parsing in dimensions.py or the NetCDF.meta_data property.
